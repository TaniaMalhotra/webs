]]> By: Daniel Tonon https://css-tricks.com/striking-a-balance-between-native-and-custom-select-elements/#comment-1757728 Sat, 13 Jun 2020 23:05:45 +0000 https://css-tricks.com/?p=312544#comment-1757728 Sorting isn’t actually a big enough change to constitute a change in context.

Change in content != change in context.

See the second example under on input.

A change of context is like going from the about page to the contact page, opening a new window, or shifting the current focus point to another element.
(See official definition for “change of context”)

If you are using a <select> as your primary navigation, that is inaccessible.

If you are just using it to sort a list of news articles immediately on change though, that is accessible.

It probably doesn’t hurt warning the user about the immediate effect though anyway.
